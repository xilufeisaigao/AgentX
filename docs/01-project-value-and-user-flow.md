# 模块 01：项目价值需求与用户交互逻辑（需求阶段）

范围说明：
本文件只包含两部分内容：
1. 项目价值需求（你明确的两个重点）
2. 用户与需求 Agent 的完整交互逻辑（到需求文档确认完成为止）

## 1. 项目价值需求（两个重点）

1. 人类绝不参与任何具体代码编写（no coding）
2. 人类只负责高价值决策或产品走向决策（决策点应显式化）

## 2. 用户完整交互逻辑（需求阶段）

1. 用户与需求 Agent 进行交互，持续补充需求与约束信息。
2. 当需求 Agent 认为当前需求信息已支持开始“初步开发”时，需求 Agent 必须询问用户：是否提交需求并开始。
3. 若用户不同意提交，需求 Agent 继续通过澄清/追问补齐信息；在再次满足“可开始初步开发”时，需求 Agent 再次询问用户是否提交需求并开始。
4. 上述询问循环持续进行，直到用户明确同意提交需求并开始。
5. 用户同意提交后，需求 Agent 输出“完整需求文档”。
6. 用户可以直接编辑需求文档或提出修改意见；需求 Agent 根据反馈修订需求文档。
7. “输出文档-用户反馈/编辑-修订文档”的循环持续进行，直到用户确认需求文档符合其需求为止。
