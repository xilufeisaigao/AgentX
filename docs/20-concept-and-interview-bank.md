# AgentX 概念与面试题复习库

这份文档用于沉淀“之后还会反复复习”的内容。

更新方式：

1. 当用户说 `结束学习` 时更新。
2. 只保留值得长期复习的概念。
3. 对用户反复追问的问题单独记录。

## 高频概念

### 微轮次学习

- 含义：
  把一次学习拆成很多个很小的轮次，每次只讲一个小 checkpoint。
- 为什么这样做：
  因为重新接手复杂项目时，理解瓶颈通常不是信息不足，而是信息过载。
- 在 AgentX 学习中的落实：
  每轮只讲 1 到 3 个函数或一个很小的类，同时配一份带教学注释的代码。

### 进度指针

- 含义：
  明确记录“当前学到哪一轮、当前 checkpoint 是什么、下一轮该讲什么”。
- 为什么这样做：
  因为你会频繁新开窗口，必须让新的 agent 无缝接力。

## 用户反复追问池

### Q1. 为什么不能一次把整个模块讲完？

- 因为那样很容易看似讲了很多，实际没有形成稳定理解。
- 对复杂系统，更有效的是：
  先目标
  再流程图
  再概念
  再少量代码
  再停下来确认理解

### Q2. 为什么要先讲基础概念，再讲项目代码？

- 因为很多项目代码的难点并不在语法，而在它依赖的背景概念。
- 如果不先补概念，代码解释会变成机械翻译。

## 复习面试题池

1. 重新接手一个复杂 agent 项目时，为什么要先建立学习进度指针？
2. 为什么复杂系统的学习更适合“流程图 + 少量代码 + 追问式下钻”的节奏？
3. 什么情况下应该先解释 Docker / DAG / RAG / Scheduler 这类概念，再进入源码？
